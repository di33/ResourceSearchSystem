# 架构审查建议

## 索引一致性
- 定义 `resources.json` 的 schema，字段包括：
  - `id`: 唯一标识符（如文件 hash 或 UUID）。
  - `source_path`: 原始资源路径。
  - `stored_path`: 拷贝后的资源路径。
  - `preview_paths`: 生成的预览文件路径列表。
  - `status`: 任务状态（如 queued, processing, success, failed）。
  - `checksum`: 文件校验值。
  - `created_at` 和 `updated_at`: 时间戳。
  - `error_msg`: 错误信息。

## 任务队列与 Worker
- 使用消息队列（如 Redis + RQ 或 Celery）管理任务：
  - 主进程扫描资源并将任务加入队列。
  - Worker 从队列中拉取任务，执行处理逻辑。
  - 任务完成后更新索引状态。

## 容器化工具
- 将 Blender 和 FFmpeg 等工具封装进 Docker 容器：
  - 确保一致的运行环境。
  - 隔离资源密集型任务，避免影响主机性能。

## 可观察性与日志
- 为每个任务生成唯一 trace_id，记录处理日志。
- 上报处理时长、成功/失败率、错误分类等指标。
- 设置失败任务的报警机制。

## 并发与资源控制
- 使用 Worker Pool 控制并发数。
- 对模型渲染等资源密集型任务设置单独队列，限制并发数。

## 安全与沙箱
- 对外部工具调用使用容器或 chroot 隔离。
- 校验输入路径和文件名，防止路径遍历攻击。

## 测试与 CI
- 增加单元测试和端到端测试覆盖率。
- 在 CI/CD 流水线中运行测试，确保代码质量。
